This notebook contains the code for running my computational test of Gregory Johnson's (1978, 1982) scalar stress theory using DeGroot opinion dynamics on structured networks. The core finding is that Johnson's O(N²) coordination cost prediction depends on an implicit complete-graph communication assumption. Under realistic fixed-degree topologies, consensus time scales far more favorably, and hierarchy's advantage over flat networks is contingent on influence heterogeneity rather than structurally inevitable.

## Overview

The simulation compares consensus time (steps to reach opinion convergence under DeGroot dynamics) across three network topologies and three influence weight distributions, yielding a 3×3 factorial design:

**Topologies** (all matched at mean degree C = 6):
- **Random Regular** (control): every node has exactly degree 6
- **Watts-Strogatz Small-World** (alternative): k = 6, p_rewire = 0.1
- **Augmented B-ary Tree** (hierarchy): branching factor b = 5, with probabilistic leaf-to-leaf edges added to reach mean degree ≈ 6

**Influence weight distributions:**
- **Equal**: all nodes weight 1.0
- **Uniform**: U(1, 10)
- **Skewed**: Lognormal(μ = 0, σ = 1.2)

For hierarchical graphs with stochastic weight distributions, weights are drawn i.i.d. from the target distribution and then sort-assigned so that higher-level nodes receive higher values. This preserves the marginal distribution while correlating influence with structural position.

Group sizes range from ~10 to ~10,000 (constrained to valid b-ary tree sizes), sampled at ~20 log-spaced points. Each condition runs 20 independent trials with separate RNG streams for graph construction and opinion initialization.

## Module Descriptions

### `engine.py`
Core simulation logic. `build_weight_matrix()` constructs a row-stochastic DeGroot weight matrix from an adjacency matrix and node influence weights using sparse CSR operations. `simulate_degroot()` iterates opinion updates until convergence (max−min < 1e-6) or a step cap. `compute_spectral_gap()` extracts the second-largest eigenvalue modulus via ARPACK (with dense fallback) to produce a spectral prediction of convergence time. `run_trial()` wraps all three into a single call.

### `generator.py`
Produces adjacency matrices and node weights. `generator()` dispatches to NetworkX for random regular and Watts-Strogatz graphs, and builds augmented b-ary trees arithmetically with probabilistic leaf-to-leaf edges weighted by hierarchical distance decay (parameter Pd = 0.5). `make_weights()` is a shared module-level function used by all topologies, ensuring identical weight distributions across conditions. `valid_tree_sizes()` computes the set of N values that yield well-formed b-ary trees.

### `simulation.py`
Parameterized runner that loops over topologies and group sizes. To reproduce the full dataset, run it three times with `weight_dist` set to `None`, `'uniform'`, and `'skewed'`. Results are serialized as pickle files organized by weight condition and degree.

### `analysis.ipynb`
Jupyter notebook that loads all pickle files and produces five analyses (as well as a visual aid for comparing network topologies):

1. **Scaling curves** — log-log predicted convergence time vs. N with OLS power-law exponents and 95% CIs
2. **Interaction plot** — topology × weight condition at the largest complete-coverage N
3. **Sigma threshold sweep** — lognormal σ swept from 0.01 to 2.0 at fixed N ≈ 1000; identifies crossover point where hierarchy becomes fastest (σ ≈ 0.77)
4. **Spectral validation** — scatter of spectral-predicted vs. simulated consensus time (Pearson r, Spearman ρ)
5. **Functional form comparison** — logarithmic, power-law, and linear fits with AIC model selection per cell

## Reproducing Results

### Requirements

- Python 3.10+
- numpy
- scipy
- networkx
- matplotlib
- pandas

Install with:
```bash
pip install numpy scipy networkx matplotlib pandas
```

### Step 1: Run Simulations

The simulation script must be run three times, once per weight condition. Edit the `weight_dist` parameter at the top of `simulation.py` for each run:

```python
# Run 1: equal weights
weight_dist = None

# Run 2: uniform weights
weight_dist = 'uniform'

# Run 3: skewed (lognormal) weights
weight_dist = 'skewed'
```

Then execute:
```bash
python simulation.py
```

Each run produces pickle files in `results/inf_distr/{equal,uniform,skewed}/C_6/` containing per-trial dictionaries keyed by group size N. Each trial records consensus time, convergence status, spectral gap, λ₂ modulus, predicted convergence time, final disagreement, and consensus value.

Expect the skewed run to take longer due to slower convergence at large N with highly heterogeneous weights.

### Step 2: Run Analysis

Open and run `analysis.ipynb` (or execute `analysis.py` as a script). The notebook loads all nine pickle files, constructs a unified DataFrame, and produces all figures. The sigma threshold sweep in Analysis 3 runs its own simulations inline (~19 σ values × 3 topologies × 20 trials at N ≈ 1000).

Figures are saved to `figures/` in both PNG (300 dpi) and PDF formats.

### Random Seeds

Graph construction and opinion initialization use separate, fixed RNG streams (`seed=21` and `seed=42` respectively in `simulation.py`; `seed=99` and `seed=77` for the sigma sweep in the notebook).
## How Data Are Obtained

All data are computationally generated. The pipeline is:

1. **Graph generation**: NetworkX constructs the base topology (random regular, Watts-Strogatz, or b-ary tree). For hierarchy, additional leaf-to-leaf edges are added probabilistically with weights decaying by hierarchical distance (Pd^(max_depth − depth_of_LCA)).
2. **Weight assignment**: `make_weights()` draws from the specified distribution. For hierarchy with stochastic weights, values are sort-assigned to preserve the marginal distribution while correlating influence with tree depth.
3. **Weight matrix construction**: The adjacency matrix is augmented with self-loops, columns are scaled by node influence weights, and rows are normalized to produce a row-stochastic DeGroot matrix (sparse CSR format).
4. **Opinion dynamics**: Initial opinions are drawn U(0,1). The matrix-vector product W·x is iterated until max(x)−min(x) < 1e-6 or 50,000 steps.
5. **Spectral analysis**: The two largest eigenvalues of W are computed via ARPACK. The spectral gap (1−|λ₂|) yields a predicted convergence time: log(1e-6) / log(|λ₂|).
6. **Serialization**: Per-trial results are stored in nested dictionaries (`{N: [list of trial dicts]}`) and pickled.
